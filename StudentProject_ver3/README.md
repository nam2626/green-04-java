# StudentProject_ver3

학적관리 프로그램 Version 3입니다.

## 프로젝트 개요
이 프로젝트는 이전 버전(StudentProject_ver2)의 구조를 더욱 발전시켜, 디자인 패턴을 적용한 객체 지향 프로그래밍 연습을 목적으로 합니다.
사용자는 학생 정보를 등록, 삭제, 수정, 조회 및 전체 조회를 할 수 있습니다.

## 주요 설계 특징 및 적용된 기술
1. **Controller 패턴 적용**: 각 기능별(등록, 삭제, 수정 등)로 컨트롤러 클래스를 분리하여 단일 책임 원칙(SRP)을 준수하고 유지보수성을 높였습니다.
2. **인터페이스 기반 다형성**: `Controller` 인터페이스를 정의하고, 모든 구체적인 컨트롤러가 이를 구현하게 함으로써 실행 흐름을 통일했습니다.
3. **HandlerMapping (팩토리 패턴 변형)**: 사용자 입력 번호에 따라 적절한 컨트롤러를 생성하여 반환하는 로직을 분리하였습니다.
4. **싱글톤(Singleton) 패턴**:
   - `StudentService`: 데이터 저장소(ArrayList)를 관리하며 프로그램 전체에서 하나의 인스턴스만 유지합니다.
   - `HandlerMapping`: 매핑 정보 관리 객체를 하나만 유지합니다.
5. **Collection Framework**: `ArrayList`를 사용하여 가변적인 학생 데이터를 관리합니다.

## 패키지 구조
- `main`: 프로그램의 진입점 (`StudentMain`)
- `controller`: 사용자 요청 처리를 위한 인터페이스 및 구현체들
- `service`: 비즈니스 로직 및 데이터 관리 (`StudentService`)
- `vo`: 학생 데이터를 담는 객체 (`StudentVO`)

## 실행 방법
`main.StudentMain` 클래스를 실행하면 콘솔 메뉴를 통해 프로그램을 이용할 수 있습니다.
