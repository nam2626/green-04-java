# 객체 지향 프로그래밍 (Object Oriented Programming)

초보자를 위한 자바 OOP 핵심 정리입니다. 현재까지 학습한 **클래스, 생성자, 상속**의 핵심 개념을 담고 있습니다.

---

## 1. 클래스(Class)와 객체(Object)
- **클래스**: 객체를 만들기 위한 **'설계도'** 혹은 **'붕어빵 틀'**입니다. 
  - 예: `Person` 클래스 (사람이라는 개념의 설계도)
- **객체(인스턴스)**: 설계도를 바탕으로 **메모리에 실제로 만들어진 실체**입니다.
  - 예: `p1`, `p2` (실제로 숨 쉬고 이름이 있는 개별적인 사람)

### 클래스의 구성 요소
1. **필드(Field)**: 객체의 **상태나 특징** (변수)
   - 예: 이름, 나이, 학번, 급여
2. **메서드(Method)**: 객체가 하는 **동작이나 기능** (함수)
   - 예: 밥 먹기, 정보 출력하기, 성적 계산하기
3. **생성자(Constructor)**: 객체가 처음 태어날 때 **초기 설정을 담당**하는 특수한 메서드

---

## 2. 생성자 (Constructor)
객체를 생성(`new`)할 때 반드시 호출되며, 주로 필드 값을 초기화합니다.
- **특징**: 클래스 이름과 똑같아야 하며, 반환 타입(void 등)이 없습니다.
- **this 키워드**: "이 객체 자신"을 의미합니다. 필드 이름과 매개변수 이름이 같을 때 구분하기 위해 사용합니다.
  ```java
  public Person(String name) {
      this.name = name; // 내 필드 name에 받은 name을 넣는다!
  }
  ```

---

## 3. 정보 은닉과 접근 제어자
데이터를 외부에서 함부로 바꾸지 못하게 보호하는 기능입니다.
1. **private**: **나만 사용** (클래스 내부만) -> 보통 필드(변수)에 사용
2. **public**: **누구나 사용** -> 보통 메서드나 생성자에 사용
3. **protected**: 상속받은 자식 클래스까지만 허용
4. **default**: 같은 패키지(폴더) 내에서만 허용

---

## 4. 상속 (Inheritance)
부모 클래스의 기능을 자식 클래스가 그대로 물려받는 것입니다. (`extends` 사용)
- **재사용성**: 부모의 코드를 다시 쓸 필요가 없습니다.
- **메서드 오버라이딩(Overriding)**: 부모에게 물려받은 기능을 자식의 상황에 맞게 **재정의**하는 것입니다.
  - 예: `Animal`의 `eat()` (동물이 먹는다) -> `Human`의 `eat()` (사람이 밥을 먹는다)

---

## 5. 다형성 (Polymorphism)
하나의 타입(부모)으로 여러 가지 형태(자식들)를 다룰 수 있는 능력입니다.
- **업캐스팅(Upcasting)**: 자식 객체를 부모 타입의 변수에 담을 수 있습니다.
  ```java
  Animal a = new Human(); // 사람은 동물이다 (가능!)
  a.eat(); // 실행하면 '사람이 밥을 먹습니다'가 출력됨 (다형성)
  ```
- **장점**: 코드 하나로 여러 종류의 객체를 일괄 관리할 수 있어 확장이 매우 편리합니다.
