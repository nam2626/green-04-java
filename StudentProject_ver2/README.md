# 학적 관리 프로그램 Version 2 (Refactoring Project)

이 프로젝트는 `StudentProject_ver1`의 단순한 구조를 객체 지향의 핵심 원칙을 적용하여 더 체계적으로 업그레이드한 버전입니다.

---

## 🚀 왜 구조를 바꿨을까요? (ver1 vs ver2)

### **기존 구조 (ver1)의 문제점**
- **Main 클래스의 과부하**: `StudentMain`이 메뉴 출력, 입력 받기, 어떤 기능을 실행할지 결정하는 일까지 너무 많은 일을 혼자 처리했습니다.
- **유지보수의 어려움**: 기능이 100개로 늘어난다면 `Main` 클래스의 `switch-case`문은 수천 줄이 되어 코드를 읽기조차 힘들어집니다.
- **분업의 부재**: 요리사 한 명이 주문도 받고, 요리도 하고, 서빙도 하고, 계산도 하는 상황과 같습니다.

### **새로운 구조 (ver2)의 도입**
- **역할 분담**: 각 클래스가 "딱 한 가지 종류의 일"만 잘하도록 쪼개어 효율성을 높였습니다.

---

## 📂 핵심 구성 요소와 역할

### 1. **Controller (컨트롤러) : "분야별 전문가"**
- **역할**: 사용자의 요청(메뉴 선택) 하나하나를 전담해서 처리합니다.
- **왜 필요한가?**: 등록 전문가(`StudentInsertController`), 조회 전문가(`StudentAllPrintController`) 등으로 나누어 놓으면, 나중에 등록 기능에 버그가 생겼을 때 '등록 전문가' 코드만 열어보면 됩니다. 다른 코드를 건드릴 위험이 사라집니다.

### 2. **HandlerMapping (핸들러 매핑) : "지배인/안내원"**
- **역할**: 사용자가 입력한 번호를 보고 "그 일은 이 전문가(Controller)가 담당이야!"라고 연결해 줍니다.
- **왜 필요한가?**: `Main` 클래스가 일일이 전문가를 찾을 필요 없이, 안내원(`HandlerMapping`)에게 물어보기만 하면 전문가를 데려다주기 때문에 `Main` 코드가 매우 깔끔해집니다.

### 3. **StudentService (싱글톤 서비스) : "중앙 데이터 창고"**
- **역할**: 모든 학생 데이터가 저장된 배열을 관리합니다.
- **특징**: `Singleton` 패턴을 사용하여 프로그램 전체에서 **딱 하나의 창고**만 존재하도록 보장합니다. 등록 전문가든 조회 전문가든 모두 같은 창고를 이용해야 데이터가 꼬이지 않기 때문입니다.

---

## 💡 비유로 이해하기 (음식점)

- **StudentMain**: 식당의 전체적인 영업 시간(무한 루프)을 관리합니다.
- **HandlerMapping**: 손님이 "스테이크 주세요"라고 하면 "양식 담당 요리사"를 호출해 주는 **주방 지배인**입니다.
- **Controller**: 지배인의 호출을 받아 실제 요리를 완성하는 **각 파트 요리사**입니다.
- **StudentService**: 요리사들이 공통으로 재료를 꺼내 쓰고 저장하는 **대형 냉장고**입니다.

---

## 📈 발전 방향
현재 등록(1번)과 전체 조회(5번) 기능이 컨트롤러 패턴으로 전환되었습니다. 나머지 기능(삭제, 수정 등)도 순차적으로 전담 전문가(Controller)를 만들어 독립시킬 예정입니다.

---

## 🏫 패키지 및 클래스별 역할 상세 (음식점 비유)

### 1. `main` 패키지 : 🏁 식당의 지배인 (매니저)
*   **`StudentMain.java`**
    *   **역할**: 프로그램의 전체적인 운영을 담당하는 **홀 지배인**입니다.
    *   **기능**: 손님(사용자)에게 메뉴판을 보여주고 주문을 받습니다. 직접 요리를 하지는 않고, 주문이 들어오면 주방 안내원에게 "누구를 불러올지" 물어봅니다.

### 2. `controller` 패키지 : 👨‍🍳 파트별 요리사 (기능 담당)
*   **`Controller.java` (부모 인터페이스)**
    *   **역할**: 모든 요리사가 지켜야 할 **공통 규격**입니다. "모든 요리사는 요리 시작(execute) 명령을 받으면 일을 시작한다"라는 약속입니다.
*   **`HandlerMapping.java`**
    *   **역할**: 주문을 보고 적절한 요리사를 연결해 주는 **주방 안내원**입니다.
    *   **기능**: "1번 주문이면 등록 요리사에게, 5번이면 출력 요리사에게 가세요"라고 안내해 줍니다.
*   **`StudentInsertController.java` 등 (개별 컨트롤러)**
    *   **역할**: 각 기능(등록, 삭제, 수정 등)을 전담하는 **파트별 전문 요리사**입니다. 자기 담당 메뉴가 들어왔을 때만 움직입니다.

### 3. `service` 패키지 : 📦 데이터 창고지기 (핵심 로직)
*   **`StudentService.java`**
    *   **역할**: 식재료(데이터)가 보관된 **대형 냉장고와 창고지기**입니다.
    *   **특징**: **싱글톤(Singleton)** 패턴을 사용하여 식당에 단 하나의 냉장고만 존재하게 합니다. 어느 요리사가 오든 똑같은 냉장고를 써야 데이터가 꼬이지 않기 때문입니다.

### 4. `vo` 패키지 : 📄 학생 정보 양식 (데이터 단위)
*   **`StudentVO.java` (Value Object)**
    *   **역할**: 학생 한 명의 정보를 적어두는 **표준 주문서**입니다.
    *   **기능**: 학번, 이름, 학과 등 흩어져 있는 데이터를 하나로 깔끔하게 묶어주는 박스 역할을 합니다.

## 🔄 프로그램 동작 흐름 (한눈에 보기)

1.  **사용자**: "5번(전체 조회) 선택!"
2.  **StudentMain (지배인)**: `HandlerMapping`에게 "5번 담당 요리사 좀 데려와!"라고 합니다.
3.  **HandlerMapping (안내원)**: "네, `StudentAllPrintController` 요리사님 호출합니다."
4.  **Controller (요리사)**: `execute()` 명령을 받아 일을 시작합니다. 이때 `StudentService` 창고지기에게 가서 "학생 목록 좀 다 꺼내주세요"라고 요청합니다.
5.  **StudentService (창고지기)**: 창고(배열)에서 학생 데이터를 꺼내 요리사에게 전달합니다.
6.  **결과**: 요리사가 화면에 학생 정보를 예쁘게 출력합니다.

## 🛠 상세 실행 과정 (코드 흐름 따라가기)

초보자분들은 아래 순서대로 코드가 어떻게 움직이는지 살펴보세요!

### **1단계: 메뉴 입력 (Waiting for Order)**
*   `StudentMain` 클래스의 `while(true)` 루프가 돌아가며 사용자에게 메뉴 번호를 입력받습니다.
*   예: 사용자가 `1`을 입력하면 `no` 변수에 `1`이 저장됩니다.

### **2단계: 전문가 찾기 (Mapping Phase)**
*   `StudentMain`은 직접 "1번이 뭐였지?"라고 고민하지 않습니다. 
*   대신 `HandlerMapping.getInstance().createController(no)`를 호출하여 **"1번 담당자(Controller)를 데려와줘"**라고 요청합니다.
*   `HandlerMapping`은 `switch-case` 문을 통해 `1`번에 해당하는 `StudentInsertController` 객체를 생성해서 돌려줍니다.

### **3단계: 임무 수행 (Execution Phase)**
*   데려온 담당자(`controller`)가 `null`이 아니라면, `controller.execute(sc)` 명령을 내립니다.
*   이때 **다형성(Polymorphism)**의 마법이 일어납니다! 부모인 `Controller` 타입으로 받았지만, 실제로는 `StudentInsertController`의 `execute`가 실행됩니다.

### **4단계: 데이터 보관 및 조회 (Service Interaction)**
*   각 `Controller`는 작업을 위해 학생 데이터가 필요하면 `StudentService.getInstance()`를 호출합니다.
*   **싱글톤(Singleton)** 덕분에 프로그램 어디서든 항상 **똑같은 데이터 창고**에 접근하게 됩니다. 
*   요리사(Controller)가 재료(데이터)를 냉장고(Service)에 넣거나 빼는 과정입니다.

### **5단계: 결과 보고 (Response)**
*   모든 작업이 끝나면 화면에 "등록이 완료되었습니다" 같은 메시지를 띄우고, 다시 지배인(`StudentMain`)의 메뉴 화면으로 돌아가 다음 주문을 기다립니다.
